# 🧹 Data Cleaning Portfolio

A collection of hands-on data cleaning exercises completed as part of 
[Kaggle Learn's Data Cleaning course](https://www.kaggle.com/learn/data-cleaning), 
applied to real-world datasets including Kickstarter campaigns, earthquake records, 
and police killing statistics in the US.

## 📌 About

This repository documents my journey learning core data cleaning techniques using 
Python and Pandas — from handling messy real-world data to preparing it for analysis 
and machine learning.

## 📂 Notebooks

| Notebook | Topic | Key Techniques |
|---|---|---|
| `exercise-handling-missing-values.ipynb` | Missing Values | Detecting & imputing missing data |
| `exercise-scaling-and-normalization.ipynb` | Scaling & Normalization | Min-Max scaling, Box-Cox transformation |
| `exercise-parsing-dates.ipynb` | Date Parsing | Standardizing inconsistent date formats |
| `exercise-character-encodings.ipynb` | Character Encodings | Fixing UTF-8/Windows-1252 encoding errors |
| `exercise-inconsistent-data-entry.ipynb` | Text Standardization | Fuzzy string matching with FuzzyWuzzy |

## 🚀 Key Takeaways

- Real-world data is messy — missing values, inconsistent formats, and typos are the norm, not the exception.
- Choosing the right cleaning strategy (imputation vs. removal, scaling vs. normalization) depends on the data's context and downstream use.
- Fuzzy matching is a powerful tool for standardizing human-entered text data.
